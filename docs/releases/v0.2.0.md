# Codinfy Agent Monitor v0.2.0

“Node Operations & Safe Update Center” adds live developer-runtime observability without weakening Codinfy’s local-first safety model.

## Highlights

- Live Node/Bun/Deno server, port, project, CPU, memory, uptime, orphan, conflict, and protection inventory.
- Confirmed graceful-first process control; Force Kill needs a failed first attempt and a second confirmation.
- MCP Update Center with official GitHub release checks, SemVer, notes, preflight, backup, install, rollback, and auto-install OFF.
- Project Process Map, Resource Guard, MCP Health Doctor, Session Recovery, Backup & Restore, and Notifications.
- 30 dashboard routes, 66 MCP tools, and expanded CLI command families.

## Upgrade

```bash
git pull --ff-only
corepack enable
pnpm install --frozen-lockfile
pnpm build
pnpm test
```

Run `codinfy-agent-monitor update preflight` before a confirmed package update. Existing `.codinfy-agent-monitor` data is migrated through schema defaults; no provider credential is required.

## Safety notes

- Review detected process identity before stopping anything.
- Protected/unidentified processes are inspect-only.
- Update install, rollback, and backup restore are never automatic.
- Run `monitor.review_before_commit` before committing.
